# v0.1 Acceptance

Manual checklist before tagging v0.1. Each item needs a visual capture (screenshot or short clip) attached to the release.

## Build compatibility
- [ ] Export a build from upstream PoB-PoE2 (Witch with ~80 allocated points). Import into our app. Class and allocated nodes render correctly.
- [ ] Allocate 3 additional nodes in our app. Save. Re-open in upstream PoB. New allocations are present; no other fields lost.

## Tree fidelity (positions)
- [ ] Open the same build (Witch reference) in both apps at the same zoom level. Screenshot side by side. Node positions within ~1 px.
- [ ] Repeat for Mercenary and Ranger reference builds.

## Interaction
- [ ] Pan with drag; wheel zoom; pinch zoom (trackpad if available). No jank on a mid-range laptop.
- [ ] Hover an unallocated node far from the allocation. Pathing preview appears.
- [ ] Click the hovered node. Whole path is allocated.
- [ ] Click an allocated leaf. It deallocates.

## Sidecar
- [ ] Status pill shows green with LuaJIT version on launch.
- [ ] Delete `luajit-*.exe` from binaries temporarily → app still runs; status pill shows red. Restore it and re-launch → green.
